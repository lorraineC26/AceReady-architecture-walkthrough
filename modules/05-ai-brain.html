The AI That Grades You

How a voice transcript becomes a structured scorecard — and the Gemini prompt engineering behind it.

The Customs Officer with a Checklist

When you cross a border, the customs officer does not accept a verbal story — they need your passport filled out in a specific format. Gemini works the same way: you do not send it a transcript and hope for a nice answer. You give it a strict form (a schema) it must fill in exactly.

Gemini has two jobs in AceReady. Both use the same language model, but in completely different ways.

📝
Job 1: Generate Questions

Before the interview, Gemini receives the role, level, and tech stack. It returns a JSON array of interview questions — formatted for voice (no special characters).

🎯
Job 2: Grade the Transcript

After the interview, Gemini receives the full conversation transcript. It returns a structured output JSON scorecard across 5 categories.

💡

The same AI model doing two completely different things based on two different prompts. The prompt IS the program. This is why prompt engineering is a real skill — small changes to the prompt dramatically change the output.

Telling Gemini What to Ask

The question generator uses a simple text prompt with placeholders filled in at runtime. Notice the special instruction at the end — interview questions must be voice-friendly.

app/api/vapi/generate/route.ts const { text: questions } = await generateText({ model: google("gemini-2.5-flash-lite"), prompt: `Prepare questions for a job interview. The job role is ${role}. The job experience level is ${level}. The tech stack used in the job is: ${techstack}. The focus between behavioral and technical questions should lean towards: ${type}. The amount of questions required is: ${amount}. Please return only the questions, without any additional text. The questions are going to be read by a voice assistant so do not use "/" or "*" or any other special characters which might break the voice assistant. Return the questions formatted like this: ["Question 1", "Question 2", "Question 3"] Thank you! <3 `, });
Plain English
Call Gemini and wait for the response...
Use the lightweight flash model — fast and cheap for simple text tasks...
Build the prompt by filling in the user choices: role and level...
Continue filling in the tech stack and question type (behavioral vs technical)...
Specify how many questions to generate...
Tell Gemini to skip explanatory text — just return the raw questions...
Voice assistants cannot say slash or asterisk — forbid them in the output...
Require the questions as a JSON array so the code can parse them reliably...
💡

The "Thank you! <3" at the end of the prompt is not just politeness — research shows that politeness markers in prompts can slightly improve AI output quality. It also makes the codebase more charming.

The feedbackSchema: Gemini's Report Card Template

For feedback generation, a free-form text response is not enough. The UI needs to display exact scores, category names, and improvement tips. That requires a contract — a schema.

constants/index.ts export const feedbackSchema = z.object({ totalScore: z.number(), categoryScores: z.array( z.object({ name: z.string(), score: z.number(), comment: z.string(), }), ), strengths: z.array(z.string()), areasForImprovement: z.array(z.string()), finalAssessment: z.string(), });

It must have a totalScore — a single number (0-100)...
It must have categoryScores — an array of score objects...
Each score object needs a name (the category), a score (number), and a comment (string)...
